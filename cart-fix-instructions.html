What was the problem?
Different pages were creating cart items with different data structures:
  • Product page used: { _id, id, name, price, image }
  • Women's page used: { productId, name, price, image }
  • Wishlist used: { productId, name, price, image }
The checkout validation couldn't find the product ID because it was stored in different fields. This has now been standardized.
Fixed! All pages now create cart items with consistent structure: { _id, id, productId, name, price, image }
